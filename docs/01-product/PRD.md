# PRD: 중학생 학업 성취 대시보드

- Version: v1.3
- Date: 2026-03-14
- Product Stage: Private beta content authoring implemented

## 1. 배경과 목적

대한민국 중학생이 현재 수학 학업 성취 상태를 스스로 점검하고, 약점을 인식해 보완하는 메타인지 중심 대시보드 웹서비스를 제공한다.

## 2. 목표

- 현재 날짜 기준 학년/학기 진도와 개인 성취도를 한눈에 제공
- 오답을 축적하여 단원별 취약점을 시각적으로 분석
- 중학생 친화 UI/UX 제공
- 1단계 수학 MVP 후 과목 확장 가능 구조 확보
- 중학교 시작, 초등/고등 확장 가능한 데이터 구조 적용

## 3. 사용자

- 1차 사용자: 보호자 + 중학생 1명(딸)
- 2차 사용자: 다수 학생 + 보호자

### 3.1 Current Account Model

- 보호자는 `/signup`에서 직접 가입하고 공용 로그인 페이지(`/login`)로 진입한다.
- 학생은 보호자가 발급한 초대코드로 `/student/activate`에서 첫 계정을 활성화한다.
- 보호자는 `/dashboard`, 학생은 `/student/dashboard`를 기본 진입점으로 사용한다.
- 보호자 전용 입력/관리 기능(`/students/manage`, `/records/new`, `/wrong-answers/manage`)은 학생 역할에서 접근할 수 없다.

## 4. 범위

### In Scope (MVP)

- 수학 과목만 제공
- 학년/학기/단원 진도표 조회
- 문제집/학습지별 정오답 입력
- 오답 문제 이미지 업로드
- 랜딩 페이지 + 보호자 회원가입
- 학생 초대코드 발급 + 학생 첫 활성화
- 보호자 대시보드 + 학생 대시보드 분리
- 학생 관리 화면(학생 프로필 생성/계정 상태/초대코드 발급)
- 학생 학습 런치패드(`/student/dashboard`)
- 내장 문제세트 학습 세션(`/student/study/session`)
- 단원 상태형 진도/개념 보드(`/student/progress`)
- 학생 오답노트 self-service(`/student/wrong-answers`)
- 보호자 학습 리뷰 큐(`/study/reviews`)
- 데일리 미션 추천 + skill tag 기반 연습 세트 노출
- 웹 필기 캔버스 PNG 스냅샷 저장
- 보호자/관리자 학습 콘텐츠 authoring(`/study/content`)
  - 연습 세트 생성/수정/활성화
  - 개념 자료 생성/수정/삭제
- 오답 카테고리 분류
  - 단순 연산 실수
  - 문제 잘못 읽음
  - 문제 이해 못함
- 대시보드 시각화
  - 진도 대비 성취도
  - 단원별 약점
  - 주간/월간 변화

### Out of Scope (MVP 제외)

- 실시간 학교 LMS 연동
- 자동 채점 OCR 고도화
- 자유형 필기 인식/서술형 자동 채점
- 다과목 동시 정교 분석
- 다운로드/공유 가능한 리포트 산출물
- 약점 우선순위 추천 메시지

## 5. 핵심 사용자 시나리오

1. 보호자가 회원가입 후 첫 학생 프로필을 생성한다.
2. 보호자가 학생 초대코드를 발급하고 학생은 `/student/activate`에서 첫 로그인을 완료한다.
3. 보호자는 문제집/시도/문항/오답을 입력하고 관리한다.
4. 보호자/관리자는 `/study/content`에서 현재 학기 또는 준비 중인 학기의 연습 세트와 개념 자료를 수정한다.
5. 학생은 자기 전용 대시보드에서 오늘의 미션, 최근 피드백, 오답 복습 진입점을 확인한다.
6. 학생은 개념 자료를 보고 내장 문제세트 학습 세션을 시작하고, 타이머/캔버스로 풀이를 남긴다.
7. 학생은 제출 후 생성된 오답노트에 카테고리와 자기 메모를 정리한다.
8. 보호자는 학습 리뷰 큐에서 학생 제출을 검토하고 피드백/단원 상태를 기록한다.
9. 보호자는 보호자 대시보드에서 약점 단원, 오류 유형 비중, 주간/월간 변화와 학생 학습 루프 진행 상태를 함께 확인한다.

## 6. 기능 요구사항

- FR-001: 보호자 회원가입, 학생 프로필, 학생 초대/활성화 관리
- FR-002: 교육과정(학교급/학년/학기/과목/단원) 조회
- FR-003: 문제집/학습지 등록 및 단원 맵핑
- FR-004: 시도(Attempt)별 정오답 입력
- FR-005: 오답 이미지 업로드 및 조회
- FR-006: 오답 카테고리(복수 선택 가능) 저장
- FR-007: 대시보드 시각화(진도, 성취도, 약점)
- FR-008: 기간 필터 기반 주간/월간 추이 조회
- FR-009: 학생 내장 문제세트 학습 세션 시작/제출
- FR-010: 학생 개념 자료 조회 + 단원 상태형 진도 보드
- FR-011: 보호자 학습 리뷰 큐와 피드백 저장
- FR-012: 학생 오답노트 self-service(메모/카테고리/사진 업로드)
- FR-013: 보호자/관리자 학습 콘텐츠 authoring(연습 세트/개념 자료)

### 6.1 수용 기준(AC)

- FR-001: 보호자가 회원가입 후 학생 프로필을 생성/조회할 수 있어야 하고, 학생은 초대코드로만 활성화되어야 하며, 타 보호자 학생은 조회 불가해야 한다.
- FR-002: 교육과정 조회 시 `asOfDate` 기준으로 적용 버전을 선택해 반환해야 한다.
- FR-003: 문제집 등록 시 학생 소유권 검증이 통과되어야 저장된다.
- FR-004: 시도/문항 저장 시 학생-문제집 관계와 소유권 검증이 통과되어야 저장된다.
- FR-005: 오답 이미지 업로드는 허용 포맷/용량 제한을 만족해야 저장된다.
- FR-006: 오답 카테고리는 복수 선택 가능해야 하며 조회 시 동일하게 재현되어야 한다.
- FR-007: 대시보드는 최신 저장 데이터 기준으로 진도/약점/성취도를 표시해야 한다.
- FR-008: 주간/월간 추이는 기간 필터 기준 합계/비율이 일관되어야 한다.
- FR-009: 학생은 자기 계정으로만 학습 세션을 시작/제출할 수 있어야 하고, 제출 시 자동채점과 오답 생성이 일관되어야 한다.
- FR-010: 학생은 현재 학기 단원 상태와 연결된 개념 자료/연습 세트를 한 화면에서 확인할 수 있어야 한다.
- FR-011: 보호자는 자기 학생이 제출한 세션만 리뷰할 수 있어야 하며, 리뷰 후 단원 상태가 학생/보호자 보드에 반영되어야 한다.
- FR-012: 학생은 자기 오답노트만 수정/업로드할 수 있어야 하며, 보호자 오답 관리 UX와 충돌하지 않아야 한다.
- FR-013: 보호자/관리자는 최신 교육과정 버전 기준으로 연습 세트와 개념 자료를 수정할 수 있어야 하고, 사용된 연습 세트는 구조 수정 없이 메타데이터만 바꿀 수 있어야 하며, 저장 결과는 학생 보드와 학습 세션에 즉시 반영되어야 한다.

### 6.2 Deferred After MVP (M5)

- M5-001: 다운로드/공유 가능한 리포트 산출물
- M5-002: 약점 우선순위 추천(정답률, 최근성, 반복오류 반영)
- M5-003: content authoring versioning/draft/publish workflow
- M5-004: 자유형 필기 stroke replay / 공동 편집
- M5-005: OCR/서술형 자동 판정

## 7. 비기능 요구사항

- 모바일 우선 반응형 (iPhone/iPad/PC)
- 맥 미니 로컬 호스팅 우선
- 개인정보 최소 수집
- 이미지 업로드 안정성 및 백업
- 첫 화면 로딩 3초 이내 목표

## 8. KPI

- 주 3회 이상 사용률
- 오답 기록 누적 수
- 약점 단원 재학습 후 정답률 향상
- 4주 유지율

## 9. 확장성 요구사항

모든 커리큘럼/학습 데이터에 아래 축을 유지한다.

- school_level: elementary | middle | high
- subject: math (MVP), future multi-subject
- grade, semester, unit hierarchy

## 10. 배포/운영 원칙

- Phase 1: Mac mini local self-hosting
- Phase 2: 필요 시 외부 호스팅(Vercel/Render 등) 이관
- 컨테이너 기반으로 이관 비용 최소화

### 10.1 외부 접근 정책

- Phase 1에서는 로컬 네트워크 접근을 기본 정책으로 한다.
- 외부 접근이 필요한 경우에만 reverse proxy + TLS + 접근 제어를 적용한다.
- 운영 공개 전 개인정보/권한/로그 정책 점검을 완료해야 한다.
