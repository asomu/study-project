# PRD: 오답노트 전용 서비스

- Version: v1.5
- Date: 2026-03-21
- Product Stage: Wrong-note-first private beta

## 1. 배경과 목적

중학생이 틀린 문제를 사진 1장 단위로 바로 기록하고, 보호자가 누적 데이터를 바탕으로 피드백을 남길 수 있는 오답노트 전용 서비스를 제공한다.

핵심 목표는 아래 한 가지 루프를 안정적으로 만드는 것이다.

1. 학생이 틀린 문제를 찍어 올린다.
2. 단원과 오류유형을 함께 기록한다.
3. 대시보드에서 누적 통계와 카테고리별 목록을 본다.
4. 보호자가 피드백을 남긴다.
5. 학생이 다시 확인하고 복습한다.

## 2. 제품 목표

- 학생이 사진 중심으로 오답을 빠르게 누적할 수 있어야 한다.
- 오답이 `학년/학기/단원/오류유형` 기준으로 정리되어야 한다.
- 학생/보호자 모두 같은 데이터셋을 기준으로 통계와 목록을 봐야 한다.
- 보호자 피드백이 학생 상세 화면에 다시 노출되어 복습 루프를 완성해야 한다.
- 기존 `attempt / practice / wrong-answer` 흐름이 있더라도, 현재 제품의 기본 진입 경험은 오답노트 대시보드여야 한다.

## 3. 사용자

- 1차 사용자: 보호자 + 중학생 1명
- 2차 사용자: 다수 학생 + 보호자

### 3.1 계정 모델

- 보호자는 `/signup`에서 직접 가입하고 `/login`으로 로그인한다.
- 학생은 보호자가 발급한 초대코드로 `/student/activate`에서 첫 계정을 활성화한다.
- 학생 기본 진입점은 `/student/dashboard`다.
- 보호자 기본 진입점은 `/dashboard`다.

## 4. 범위

### In Scope

- 수학 과목만 지원
- 학생 오답노트 사진 업로드
- 오답 1건당 단일 오류유형 저장
- 단원 기반 분류
  - 학교급/학년은 학생 프로필 기준
  - 학기는 학생이 `1/2` 중 선택
  - 단원은 해당 학생 학년/학기 범위에서 선택
- 오류유형 분류
  - `calculation_mistake` / 단순 연산 실수
  - `misread_question` / 문제 잘못 읽음
  - `lack_of_concept` / 문제 이해 못함
- 학생 대시보드
  - 빠른 업로드 폼
  - 총 오답 수 / 최근 30일 / 피드백 완료 / 오류유형 분포
  - 상위 단원 분포
  - 필터(`학기/단원/오류유형/기간/피드백 여부`)
  - 카드 목록 + 상세 드로어
  - 학생 메모/단원/오류유형/사진 수정
  - 삭제
- 보호자 대시보드
  - 학생 선택
  - 학생과 같은 KPI/분포/필터/카드 목록
  - 상세 드로어에서 보호자 피드백 작성/수정
- 이미지 업로드 정책
  - 저장 위치: `public/uploads/wrong-notes`
  - 허용 MIME: `jpeg/png/webp/heic/heif`
  - 시그니처 검증 + 용량 제한 적용
- 소프트 삭제
- API/대시보드 통계는 `WrongNote`만 기준으로 계산

### Out of Scope

- OCR로 문제 텍스트 추출
- 자동 피드백 생성
- 재도전 상태 추적
- PDF/주간 리포트 산출물
- AI 추천 메시지
- 다과목 확장
- 기존 `WrongAnswer` 데이터 마이그레이션

## 5. 핵심 사용자 시나리오

1. 보호자가 학생 계정을 준비한다.
2. 학생이 `/student/dashboard`에서 틀린 문제 사진을 올린다.
3. 학생이 학기, 단원, 오류유형, 메모를 저장한다.
4. 학생은 대시보드에서 누적 오답 수와 자주 틀리는 단원/이유를 본다.
5. 학생은 카드 목록에서 원하는 오답을 열어 다시 확인하고 메모나 사진을 수정한다.
6. 보호자는 `/dashboard`에서 학생을 선택하고 같은 오답 데이터와 통계를 본다.
7. 보호자는 상세 드로어에 피드백을 남긴다.
8. 학생은 다시 접속해 보호자 피드백을 확인하고 복습한다.

## 6. 기능 요구사항

- FR-001: 보호자 가입, 학생 초대, 학생 활성화가 가능해야 한다.
- FR-002: 학생은 자기 계정으로만 오답노트를 생성/수정/삭제할 수 있어야 한다.
- FR-003: 오답노트 생성 시 사진, 학기, 단원, 오류유형이 필수여야 한다.
- FR-004: 단원은 학생의 학교급/학년과 선택 학기에 맞는 수학 단원만 허용해야 한다.
- FR-005: 학생 대시보드는 `WrongNote` 누적 통계와 상위 단원 분포를 보여줘야 한다.
- FR-006: 학생 대시보드는 필터 조건에 맞는 오답만 카드 목록으로 보여줘야 한다.
- FR-007: 학생은 오답 상세에서 메모, 단원, 오류유형, 사진을 수정할 수 있어야 한다.
- FR-008: 학생은 오답을 soft delete할 수 있어야 하며, 삭제된 데이터는 기본 조회/통계에서 제외되어야 한다.
- FR-009: 보호자는 자기 학생의 오답만 조회할 수 있어야 한다.
- FR-010: 보호자는 오답 상세에서 피드백을 저장/수정/삭제할 수 있어야 한다.
- FR-011: 학생 상세 화면은 최신 보호자 피드백을 다시 보여줘야 한다.
- FR-012: `/student/wrong-answers`, `/wrong-answers/manage`는 신규 대시보드 경험으로 연결되어야 한다.

### 6.1 수용 기준

- FR-001: 보호자는 자기 학생만 선택/조회 가능해야 하고, 학생은 자기 프로필 1건만 접근 가능해야 한다.
- FR-002: 학생이 다른 학생의 오답 ID로 접근하면 `403` 또는 `404`를 받아야 한다.
- FR-003: 사진이 없거나, 오류유형이 없거나, 단원이 비어 있으면 저장되지 않아야 한다.
- FR-004: 학생 학년/학기 범위 밖 단원을 선택하면 저장되지 않아야 한다.
- FR-005: KPI는 `총 오답 수`, `최근 30일`, `피드백 완료`, `오류유형별 건수`를 포함해야 한다.
- FR-006: 필터는 `semester`, `curriculumNodeId`, `reason`, `from`, `to`, `hasFeedback` 기준으로 동작해야 한다.
- FR-007: 학생 수정 후 카드/상세/통계가 같은 데이터로 다시 반영되어야 한다.
- FR-008: 삭제된 오답은 리스트, 상세, 대시보드 집계에서 제외되어야 한다.
- FR-009: 보호자는 `studentId` 기준 ownership 검증을 통과한 학생 데이터만 볼 수 있어야 한다.
- FR-010: 보호자 피드백은 빈 값 저장 시 제거할 수 있어야 하고, 저장 시 `guardianFeedbackAt`, `guardianFeedbackByUserId`가 갱신되어야 한다.
- FR-011: 학생이 다시 상세를 열면 최신 피드백 본문과 갱신일을 볼 수 있어야 한다.
- FR-012: 레거시 오답 관리 진입점은 신규 오답노트 대시보드로 리다이렉트되어야 한다.

## 7. 비기능 요구사항

- 모바일 우선 반응형
- 학생이 사진 업로드를 3초 내외 체감으로 끝낼 수 있도록 가벼운 흐름 유지
- 이미지 MIME/시그니처 검증 필수
- 기본 업로드 제한은 `UPLOAD_MAX_BYTES` 기준(기본 5MB)
- 개인정보 최소 수집
- 로컬 파일 업로드 경로 운영/백업 가능 구조 유지

## 8. KPI

- 주간 오답 업로드 건수
- 학생 1인당 누적 오답 수
- 보호자 피드백 완료율
- 상위 반복 오류유형 감소율

## 9. 운영 원칙

- 현재 제품의 기본 홈은 학생/보호자 모두 오답노트 대시보드다.
- 기존 `attempt / study / wrong-answer` 기반 기능은 코드베이스에 남아 있더라도 현재 제품의 핵심 UX나 분석 소스 오브 트루스로 사용하지 않는다.
- 새로운 분석/피드백 기능은 `WrongNote` 중심 데이터 모델 위에서만 확장한다.
