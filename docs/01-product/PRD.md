# PRD: 오답노트 + 문제집 진도 서비스

- Version: v1.9
- Date: 2026-04-03
- Product Stage: Wrong-note-first + workbook-progress private beta

## 1. 배경과 목적

초등학생 또는 중학생이 틀린 문제를 사진 1장 단위로 바로 기록하고, 보호자가 누적 데이터를 바탕으로 피드백을 남길 수 있는 오답노트 서비스를 제공한다. 여기에 학생이 실제로 푸는 문제집의 단원별 단계를 `시작전 / 진행중 / 완료`로 관리하는 문제집 진도 기능을 함께 제공한다.

핵심 목표는 아래 한 가지 루프를 안정적으로 만드는 것이다.

1. 학생이 틀린 문제를 찍어 올린다.
2. 필요하면 연결된 문제집과 단계까지 함께 기록한다.
3. 학생과 보호자가 문제집 단원 진도를 같은 데이터로 본다.
4. 대시보드에서 누적 통계와 카테고리별 목록을 본다.
5. 보호자가 피드백을 남긴다.
6. 학생이 다시 확인하고 복습한다.

## 2. 제품 목표

- 학생이 사진 중심으로 오답을 빠르게 누적할 수 있어야 한다.
- 보호자가 UI에서 문제집 템플릿을 직접 만들고 학생에게 배정할 수 있어야 한다.
- 학생과 보호자가 단원별 문제집 단계를 같은 대시보드에서 관리할 수 있어야 한다.
- 오답이 `학년/학기/단원/오류유형` 기준으로 정리되어야 한다.
- 오답이 문제집 학습에서 나온 경우 `문제집/단계` 기준으로도 다시 찾을 수 있어야 한다.
- 학생/보호자 모두 같은 데이터셋을 기준으로 통계와 목록을 봐야 한다.
- 보호자 피드백이 학생 상세 화면에 다시 노출되어 복습 루프를 완성해야 한다.
- 기존 `attempt / practice / wrong-answer` 흐름이 있더라도, 현재 제품의 기본 진입 경험은 오답노트 대시보드여야 한다.

## 3. 사용자

- 1차 사용자: 보호자 + 초등/중등 학생 1명
- 2차 사용자: 다수 학생 + 보호자

### 3.1 계정 모델

- 보호자는 `/signup`에서 직접 가입하고 `/login`으로 로그인한다.
- 학생은 보호자가 발급한 초대코드로 `/student/activate`에서 첫 계정을 활성화한다.
- 학생 기본 진입점은 `/student/dashboard`다.
- 보호자 기본 진입점은 `/dashboard`다.

## 5. 핵심 사용자 시나리오

1. 보호자가 학생 계정을 준비한다.
2. 보호자가 `/dashboard`에서 문제집 템플릿을 직접 등록하고 학생에게 배정한다.
3. 학생이 `/student/dashboard`에서 문제집을 선택하고 오늘 진행한 단원 단계 상태를 `시작전 / 진행중 / 완료`로 바꾼다.
4. 학생이 `/student/dashboard`에서 틀린 문제 사진을 올린다.
5. 학생이 대상 학년, 학기, 단원, 오류유형, 메모를 저장하고, 필요하면 문제집과 단계를 연결한다.
6. 학생은 대시보드에서 누적 오답 수와 자주 틀리는 단원/이유를 보고, 문제집 진도 matrix와 완료율도 함께 본다.
   - 대상 학년과 학기를 바꾸면 바 차트가 즉시 다시 계산된다.
7. 학생은 카드 목록에서 원하는 오답을 열어 다시 확인하고 메모나 사진을 수정한다.
8. 보호자는 `/dashboard`에서 학생을 선택하고 같은 오답 데이터와 문제집 진도 현황을 본다.
9. 보호자는 상세 드로어에 피드백을 남긴다.
10. 학생은 다시 접속해 보호자 피드백을 확인하고 복습한다.

## 6. 기능 요구사항

- FR-001: 보호자 가입, 학생 초대, 학생 활성화가 가능해야 한다.
- FR-002: 학생은 자기 계정으로만 오답노트를 생성/수정/삭제할 수 있어야 한다.
- FR-002A: 보호자는 UI에서 문제집 템플릿을 생성하고 제목/출판사/활성 상태를 관리할 수 있어야 한다.
- FR-002B: 보호자는 자기 학생에게 문제집 템플릿을 배정하고 보관/복구할 수 있어야 한다.
- FR-002C: 학생과 보호자는 같은 학생의 문제집 진도 상태를 수정할 수 있어야 한다.
- FR-003: 오답노트 생성 시 사진, 대상 학년, 학기, 단원, 오류유형이 필수여야 한다.
- FR-003A: 문제집 템플릿은 단계 목록 1개 이상을 가져야 하고, 단계 이름은 중복될 수 없어야 한다.
- FR-004: 단원은 학생의 학교급 안에서 선택한 대상 학년과 학기에 맞는 수학 단원만 허용해야 한다.
- FR-004A: 문제집 진도 대시보드는 선택 문제집의 `단원 x 단계` matrix를 만들고, 저장된 row가 없으면 `시작전`으로 보여줘야 한다.
- FR-004B: 문제집 진도 대시보드는 `전체 단계 수 / 완료 / 진행중 / 시작전 / 완료율`을 계산해야 한다.
- FR-004C: 문제집 진도 바 차트는 단원별 완료 단계 수를 보여줘야 한다.
- FR-005: 학생 대시보드는 `WrongNote` 누적 KPI와 바 차트 기반 오답 분포를 보여줘야 한다.
- FR-005A: 학생/보호자 대시보드는 공통 앱 셸에서 primary nav와 현재 모드에 맞는 section nav를 제공해야 한다.
- FR-005B: 모바일 대시보드는 같은 nav를 off-canvas drawer로 제공하고, backdrop 또는 `Esc`로 닫을 수 있어야 한다.
- FR-006: 학생 대시보드는 필터 조건에 맞는 오답만 카드 목록으로 보여줘야 한다.
- FR-007: 학생은 오답 상세에서 메모, 단원, 오류유형, 사진을 수정할 수 있어야 한다.
- FR-007A: 학생은 오답 저장/수정 시 배정된 문제집과 문제집 단계를 선택해 오답을 연결할 수 있어야 한다.
- FR-008: 학생은 오답을 soft delete할 수 있어야 하며, 삭제된 데이터는 기본 조회/통계에서 제외되어야 한다.
- FR-009: 보호자는 자기 학생의 오답만 조회할 수 있어야 한다.
- FR-010: 보호자는 오답 상세에서 피드백을 저장/수정/삭제할 수 있어야 한다.
- FR-011: 학생 상세 화면은 최신 보호자 피드백을 다시 보여줘야 한다.
- FR-012: `/student/wrong-answers`, `/wrong-answers/manage`는 신규 대시보드 경험으로 연결되어야 한다.
- FR-013: 학생/보호자 대시보드는 `그래프 기준 / 대상 학년 / 학기` 콤보박스로 단원별/오류유형별 오답 바 차트를 전환할 수 있어야 한다.

### 6.1 수용 기준

- FR-001: 보호자는 자기 학생만 선택/조회 가능해야 하고, 학생은 자기 프로필 1건만 접근 가능해야 한다.
- FR-002: 학생이 다른 학생의 오답 ID로 접근하면 `403` 또는 `404`를 받아야 한다.
- FR-002A: 보호자는 자기 계정 범위의 템플릿만 수정할 수 있어야 한다.
- FR-002B: 보호자는 자기 학생에게만 문제집을 배정할 수 있어야 한다.
- FR-002C: 학생은 자기에게 배정된 문제집만 진도 수정과 오답 연결에 사용할 수 있어야 한다.
- FR-003: 사진이 없거나, 오류유형이 없거나, 단원이 비어 있으면 저장되지 않아야 한다.
- FR-003A: 단계가 0개이거나 단계명이 중복된 템플릿은 저장되지 않아야 한다.
- FR-004: 학생 학교급 범위 밖 학년이나, 선택한 대상 학년/학기 범위 밖 단원을 선택하면 저장되지 않아야 한다.
- FR-004A: 문제집 진도 matrix는 저장 레코드가 없는 셀을 `시작전`으로 렌더링해야 한다.
- FR-004B: 셀 클릭 시 상태가 `시작전 -> 진행중 -> 완료 -> 시작전` 순서로 즉시 반영되어야 한다.
- FR-004C: 보호자와 학생이 같은 문제집을 열면 동일한 summary/bar/matrix를 봐야 한다.
- FR-005: KPI는 `총 오답 수`, `최근 30일`, `피드백 완료`, `오류유형별 건수`를 포함해야 한다.
- FR-005A: 보호자는 `대시보드`, `학생 관리` primary nav를 보고, 학생/보호자 대시보드는 현재 모드에 맞는 section nav를 보여줘야 한다.
- FR-005B: 모바일에서는 `메뉴 열기 -> drawer -> 섹션 선택` 흐름과 `Esc`/backdrop 닫기가 동작해야 한다.
- FR-006: 필터는 `grade`, `semester`, `curriculumNodeId`, `reason`, `from`, `to`, `hasFeedback` 기준으로 동작해야 한다.
- FR-007: 학생 수정 후 카드/상세/통계가 같은 데이터로 다시 반영되어야 한다.
- FR-007A: 문제집을 선택한 오답은 카드/상세에서 `문제집명 + 출판사 + 단계명`을 다시 보여줘야 한다.
- FR-008: 삭제된 오답은 리스트, 상세, 대시보드 집계에서 제외되어야 한다.
- FR-009: 보호자는 `studentId` 기준 ownership 검증을 통과한 학생 데이터만 볼 수 있어야 한다.
- FR-010: 보호자 피드백은 빈 값 저장 시 제거할 수 있어야 하고, 저장 시 `guardianFeedbackAt`, `guardianFeedbackByUserId`가 갱신되어야 한다.
- FR-011: 학생이 다시 상세를 열면 최신 피드백 본문과 갱신일을 볼 수 있어야 한다.
- FR-012: 레거시 오답 관리 진입점은 신규 오답노트 대시보드로 리다이렉트되어야 한다.
- FR-013: `단원별 오답 현황`은 선택 학년/학기의 전체 단원을 0건까지 포함해 보여줘야 하고, `오류유형별 오답 현황`은 3개 오류유형을 고정 순서로 보여줘야 한다.

## 7. 비기능 요구사항

- 모바일 우선 반응형
- desktop에서는 좌측 고정 사이드바 + 상단 sticky utility bar, 모바일에서는 off-canvas drawer를 사용한다.
- 학생이 사진 업로드를 3초 내외 체감으로 끝낼 수 있도록 가벼운 흐름 유지
- 문제집 진도 matrix는 모바일과 태블릿에서 가로 스크롤 가능해야 한다.
- 상세 드로어는 keyboard로 닫기와 포커스 순환이 가능해야 한다.
- skip link, section anchor, `aria-current`, drawer focus restore를 제공해야 한다.
- 이미지 MIME/시그니처 검증 필수
- 기본 업로드 제한은 `UPLOAD_MAX_BYTES` 기준(기본 5MB)
- 개인정보 최소 수집
- 로컬 파일 업로드 경로는 git repo 밖 앱 데이터 루트로 분리하고, `~/Library/Application Support/study-project-backups` 기준 백업 가능 구조를 유지한다.

## 9. 운영 원칙

- 현재 제품의 기본 홈은 학생/보호자 모두 오답노트 대시보드다.
- 문제집 템플릿 입력은 v1에서 보호자 UI 직접 입력으로만 지원한다.
- 문제집 단계 구조는 같은 템플릿의 모든 단원에 공통 적용한다.
- 기존 `attempt / study / wrong-answer` 기반 기능은 코드베이스에 남아 있더라도 현재 제품의 핵심 UX나 분석 소스 오브 트루스로 사용하지 않는다.
- 새로운 분석/피드백 기능은 `WrongNote`와 `Workbook` 중심 데이터 모델 위에서 확장한다.
