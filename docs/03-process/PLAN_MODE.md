# Plan Mode 운영 정의

## 목적

개발 시작 전 설계 완성도를 끌어올리고, 진행 상태를 명확히 통제한다.

## 단계

1. Discovery
- 사용자/문제/목표/KPI 정리
- PRD 확정

2. Design
- 데이터 모델, 아키텍처, API 초안
- UI 플로우/컴포넌트 원칙 확정

3. Build
- 기능 단위 구현 (우선순위 기반)
- 테스트 동반 개발

4. Verify
- 단위/통합/E2E/성능/접근성 체크
- QA 체크리스트 통과

5. Release
- 로컬 배포 안정화
- 운영 모니터링 준비

## 게이트(Go/No-Go)

- G1: PRD + 아키텍처 + 스택 + 프로젝트 구조 문서 존재
- G2: 개발 플랜 + 상태 문서 운영 시작
- G3: 테스트/리뷰 정책 합의
- G4: MVP 기능 정의 완료 후 코딩 시작

## 운영 규칙

- 모든 의사결정은 `DECISION_LOG.md`에 기록
- 진행률은 `PROJECT_STATUS.md`에서만 관리
- 새로운 기능 요청은 PRD 범위 변경 후 착수
