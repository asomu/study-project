# Review Plan

## 1. 리뷰 원칙

- 동작 안정성 > 코드 스타일
- 회귀 위험 > 신규 기능 속도
- 단순 지적보다 수정 가능한 제안 중심

## 2. 리뷰 체크리스트

- 요구사항 충족 여부
- 데이터 모델/마이그레이션 안정성
- 성능/확장성 영향
- 보안/개인정보 노출 위험
- 테스트 커버리지 적정성

## 3. 심각도 기준

- P0: 기능 중단/데이터 손실
- P1: 주요 기능 오동작
- P2: 유지보수/품질 저하
- P3: 개선 권고

## 4. 리뷰 주기

- 기능 PR마다 동료 리뷰 1회 이상
- 주 1회 아키텍처 리뷰
- 2주 1회 품질 회고

## 5. 산출물

- 리뷰 코멘트 로그
- 잔여 리스크 목록
- 다음 스프린트 개선 항목
